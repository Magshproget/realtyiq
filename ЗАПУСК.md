# RealtyIQ — Інструкція із запуску

---

## Спосіб 1 — Docker (рекомендовано, один термінал)

### Що потрібно
- [Docker Desktop](https://www.docker.com/products/docker-desktop/) — встановити і запустити

### Запуск
```bash
cd C:\Users\Maknaget\Desktop\Дипломна робота\diploma\realtyiq

docker compose up --build
```

Перший запуск займе ~3-5 хвилин (завантажує образи і встановлює залежності).  
Наступні запуски — ~20 секунд.

### Відкрити програму
Браузер: **http://localhost:5173**

### Зупинка
```bash
docker compose down
```

## Спосіб 2 — Вручну (3 термінали, без Docker)

### Що потрібно
- **Python 3.12** — [python.org](https://www.python.org)
- **Node.js 18** — [nodejs.org](https://nodejs.org)
- Залежності вже встановлені (node_modules, pip packages)

### Термінал 1 — ML-сервіс (Python FastAPI)

```bash
cd .../realtyiq
python -m uvicorn ml.predict:app --host 0.0.0.0 --port 8000
```

Очікувати: `INFO: Uvicorn running on http://0.0.0.0:8000`

### Термінал 2 — Backend (Node.js Express)

```bash
cd .../realtyiq/backend
node app.js
```

Очікувати: `RealtyIQ Backend: http://localhost:3001`

### Термінал 3 — Frontend (React + Vite)

```bash
cd .../realtyiq/frontend
npm run dev
```

Очікувати: `➜  Local: http://localhost:5173/`

### Порядок важливий

```
Термінал 1: ML (8000)  →  Термінал 2: Backend (3001)  →  Термінал 3: Frontend (5173)
```

### Зупинка
В кожному терміналі: **Ctrl+C**

## Якщо щось не запускається

### Порт зайнятий (Windows)
```powershell
netstat -ano | findstr :8000
taskkill /F /PID <PID>
```

### ML-сервіс не відповідає — відновити модель
```bash
cd .../realtyiq
python ml/train.py
```

### Docker не бачить зміни в коді
```bash
docker compose up --build
```
